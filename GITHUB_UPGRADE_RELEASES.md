# Publishing HISAB KITAB updates through GitHub

Installed copies include `Upgrade.exe` beside `HISAB KITAB.exe` under Program Files. The upgrader checks the latest GitHub Release for `paul-hbledgerpro/HISABKITABWORKS` and installs the compiled update ZIP attached to that release.

## Create a new update

1. Bump the application version in `Directory.Build.props`, for example from `1.0.72` to `1.0.73`.
2. Set the same version in `installer/InnoSetup/ManagerPaperworkSystem.iss` when producing a new full installer.
3. Build the application and update package:

   ```powershell
   powershell -NoProfile -ExecutionPolicy Bypass -File .\installer\publish_all.ps1
   ```

4. Confirm these generated files exist:

   ```text
   installer\publish\win-x64\HISAB KITAB.exe
   installer\publish\win-x64\Upgrade.exe
   installer\publish\win-x64\version.txt
   installer\release\HISAB_KITAB_Update_win-x64_<version>.zip
   ```

5. Push the source changes to GitHub.
6. On GitHub, create a Release whose tag exactly matches the application version, such as `v1.0.73`.
7. Attach the generated `HISAB_KITAB_Update_win-x64_<version>.zip` file to the Release and publish it.

Do not attach the repository source ZIP as the update package. `Upgrade.exe` only installs the compiled ZIP generated by the publish script.

## Customer update flow

1. The customer opens `Upgrade.exe` from the HISAB KITAB installation folder or the Start menu shortcut named **Check for Updates**.
2. Windows requests administrator permission because the application is installed under Program Files.
3. The upgrader checks GitHub automatically in the background.
4. When a newer release is available, the customer selects **Download & Install Update**.
5. The upgrader downloads the ZIP, closes HISAB KITAB, backs up the existing installed files under `%LOCALAPPDATA%\Hisab Kitab\update_backup`, applies the update, and offers to restart the application.

Runtime data, licenses, databases, and connection settings remain under `%LOCALAPPDATA%\Hisab Kitab` and are not included in or replaced by update packages.
